Privacy & Security Policies Privacy & Security Privacy Policy

Device and System Data types

Basic technical information is stored such as device type, operating system version, anonymous usage statistics or error logs may be collected to improve app reliability and performance.

Device Data Collection & Storage

The users device uploads data into our secure servers. This data settings remain active only while the app is open, and they are cleared when the app is closed unless the user enables persistent storage. No information is uploaded to external servers except location coordinates, which are used solely to retrieve postage data and are not saved after the request is completed.

Storage Method

All data entered into the Ap is entered into a form by the user and this data is temporarily kept in the browser memory until closed or refreshed. Some data is stored if required and can be cached for longer periods of time.

Security Risks

Risk of incorrect data entry without validation. -Risk of someone using the device seeing or modifying inventory items. If using API: risks in data transmission (e.g., interception) if not protected. Risks of weak identifiers (if item names are not unique). Local session-based data can be lost, altered, or viewed by others using the same device.

Security Controls

Input validation (required fields, numeric checks, unique ID logic).Restricted modification: items can only be edited or deleted by name.Use of HTTPS (API endpoint is HTTPS).Clear warnings and confirmation dialogs (delete confirmation).No sensitive data stored, reducing overall privacy impact.Separation between UI and data logic (clean architecture reduces errors).
